=================

- Input : Recebimento das informações iniciais que serão usadas (informação digital e componente fisico) => Transforma a informações para que ela seja entendida no processo de desenvolvimento . => Misturar os ingredientes 

- Processing : Pegar várias informações, organizá-las  baseando nas regras pré-definidas com ituito de realizar algum objetivo.  => Assar, fazendo com que a mistura aconteça e os ingredientes interajam

- Output: É a saída de dados, o resultado do programa . Há vários tipos de resultado final (textual, fisico, ações fisicas, ações digitais) => bolo pronto 

- Algoritimo, receita de bolo => Passo a passo organizado com o intuito de resolver um problema especifico/tarefa .

HTTP X HTTP X SSL 


    - HTTP : Hypertext Transfer Protocol => Protocolo (conjunto de regras) de transferencia  que habilita a comunicação entre vários sistemas 

    - HTTPS : Hypertext Transfer Protocol Secure => Protocolo (conjunto de regras) de transferencia  que habilita a comunicação entre vários sistemas porém com uma camada a mais de segurança . 

        - Utiliza chaves criptografadas para validar os dados 

        - Dados não visisveis para hackers ou ususários externos 

        - Impulsiona o raqueamento do site na pesquisa do google 

    - SSL => Secure Sockets Layer => Confirma a identidade do site e possibilita uma conexão criptografada => Cria um link cripitografado entre o servidor e o navegadoe web (CADEADO AO LADO DA URL )
